        <h1>Navigation</h1>
        <p>Pages have been rearranged or combined to form this new structure.</p>
        <h3>PRIMARY &amp; SECONDARY</h3>
        <ul>
            <li><p><strong>Numbers</strong> includes:</p>
                <ul>
                    <li><strong>My Numbers</strong>, a place to view your Bandwidth network numbers and external numbers, add or remove external numbers, and configure line features for one or many numbers</li>
                    <li><strong>Buy</strong>, a place to order new numbers, import a batch from a spreadsheet, or access past new number orders</li>
                    <li><strong>Port</strong>, a place to check portability, port numbers onto or off (LSR) the Bandwidth network, or access past port requests</li>
                    <li><strong>Disconnect</strong>, a place to disconnect one or many numbers as you can do today, and access to past disconnect orders</li>
                    <li><strong>Transaction History</strong>, a collection of all past "orders", separated by order type</li>
                    <li><strong>Groups</strong>, a place to add, edit, or delete sub-accounts (aka "sites", i.e. address groups) and locations (aka "SIP peers", i.e. IP groups)</li>
                </ul>
            </li>
            <li><p><strong>Applications</strong> includes:</p>
                <ul>
                    <li>A single page to add, edit, or delete both voice and messaging applications as they exist today in Catapult</li>
                </ul>
            </li>
            <li><p><strong>9-1-1</strong> includes Dash stuff (what exactly is that?)</p></li>
            <li><p><strong>Reports</strong> includes:</p>
                <ul>
                    <li>the same "Reports Dashboard" that exists today. "Billing Detail Records" will be moved under Account > Billing.</li>
                </ul>
            </li>
        </ul>
        <h3>UTILITY</h3>
        <ul>
            <li><p><strong>Search:</strong> a link that will reveal a full-page search experience where you can type in phone numbers, order IDs, sub-account or location names, etc. Phone number searches will bring the user to existing details pages for numbers they own or suggest it's available for purchase, porting, etc.</p></li>
            <li><p><strong>Documentation:</strong> an external link to API documentation</p></li>
            <li><p><strong>Support:</strong></p>
                <ul>
                    <li>Short term - an external link to Bandwidth Support (Zendesk)</li>
                    <li>Long term - opens a panel or full-page experience with ways to read FAQs, watch learning lab videos, live chat, etc.</li>
                </ul>
            </li>
            <li><p><strong>Account</strong> includes:</p>
                <ul>
                    <li><strong>Account Overview</strong>, </li>
                    <li><strong>Users</strong>, the new user management experience</li>
                    <li><strong>Notifications</strong>
                        <ul>
                            <li>Short term - an updated version of the existing experience</li>
                            <li>Long term - this could be moved to a new box within individual user profiles</li>
                        </ul>
                    </li>
                    <li><strong>Billing</strong>, an updated experience in the works that will contain invoices, CDRs (currently "BDR"), MDRs, Statement BDRs, DID snapshots (for DTH, or Phone Number History for TRACT), and credit card payment</li>
                    <li><strong>Trunking Settings</strong>, same as today, needs UX exploration</li>
                    <li><strong>SIP Credentials</strong>, the new experience that's near completion</li>
                </ul>
            </li>
            <li><p><strong>Log out</strong></p></li>
        </ul>
        <h3>TBD</h3>
        <ul>
            <li>We need a means to switch account ("Service Account" [SAN]) as exists today in the utility nav</li>
            <li>How will Global Account Numbers (GAN) affect this, especially billing?</li>
            <li>An Administration menu will need to exist somewhere</li>
        </ul>
